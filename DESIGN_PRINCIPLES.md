# 課程網站設計原則

## 📌 核心定位

### 網站功能
本課程網站主要功能：
1. **課程進度說明** - 每週上課內容、教材整理
2. **教材下載** - 課堂筆記 PDF、補充資料
3. **作業說明** - 完整的作業內容、要求、評分標準

### 與 TronClass 的分工

```
┌─────────────────────────────────────────┐
│          課程網站 (Quarto)               │
│  ✅ 課程進度說明                          │
│  ✅ 教材下載 (PDF)                        │
│  ✅ 作業說明（完整內容）                   │
│  ✅ 演算法式教學法介紹                     │
│  ✅ 考試範圍與日期                        │
└─────────────────────────────────────────┘
                    ↓
           學生閱讀作業說明後
                    ↓
┌─────────────────────────────────────────┐
│           TronClass 平台                 │
│  ✅ 作業繳交                              │
│  ✅ 測驗作答                              │
│  ✅ 成績查詢                              │
│  ✅ 點名紀錄                              │
└─────────────────────────────────────────┘
```

## 🎯 設計原則

### 原則 1：說明在網站，繳交在 TronClass

**❌ 錯誤做法**：
- 網站上只寫「請至 TronClass 查看」
- 學生需要在兩個平台間來回查看

**✅ 正確做法**：
- 網站提供**完整詳細的作業說明**
- 包含：題型、字數、評分標準、注意事項
- 但明確標示：**繳交請至 TronClass**

### 原則 2：網站作為「公開參考資料」

- 網站連結會貼在 TronClass 公告
- 學生可以隨時查閱，不需登入
- 方便學生事前預習、事後複習

### 原則 3：TronClass 作為「互動平台」

- 實際作業繳交
- 成績管理
- 師生互動

## 📝 內容編寫範例

### ✅ 好的作業說明（網站上）

```markdown
## 課後作業一

- **測驗形式**：線上測驗（個人測驗）
- **題目類型**：3 道單選題 + 5 道是非題 + 1 道簡答題（共 100 分）
- **開放時間**：2025/10/09 21:00
- **截止日期**：2025/10/23 19:14
- **配分**：8%
- **作答次數**：最多 10 次（採最後一次得分）
- **繳交方式**：TronClass 線上測驗 ⬅️ 明確標示

:::{.callout-tip}
## 提醒事項
- ✅ 可以彼此互相討論
- ✅ 繳交次數不限（最多 10 次）
- ⚠️ 截止後會立即公布答案，**不接受遲交**
- 📝 **請至 TronClass 作答** ⬅️ 再次提醒
:::
```

### ❌ 不好的作業說明（過於簡略）

```markdown
## 課後作業一

請至 TronClass 查看作業詳情並繳交。
```

## 🔄 內容更新流程

1. **TronClass 公布作業** → 取得作業詳細資訊
2. **更新網站內容** → 將完整說明寫入 `assignments.qmd`
3. **部署網站** → `make build && make deploy`
4. **TronClass 貼連結** → 在公告中附上網站連結

## 📋 檢查清單

更新作業資訊時，確認：
- [ ] 作業說明夠詳細（題型、字數、評分標準）
- [ ] 明確標示「繳交方式：TronClass」
- [ ] 提供截止日期和配分
- [ ] 如有特殊規則，使用 callout 區塊強調
- [ ] 不要只寫「請至 TronClass 查看」
